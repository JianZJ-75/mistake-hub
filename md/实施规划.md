### 迭代 3：艾宾浩斯复习调度算法（第 5~6 周）

**目标**：实现核心调度算法，自动生成每日复习任务。

| # | 任务 | 说明 |
|---|------|------|
| 3.1 | 设计并建表 `review_plan` | 字段：id、account_id、mistake_id、planned_date、status（pending/completed/skipped）、created_time |
| 3.2 | 设计并建表 `review_record` | 字段：id、account_id、mistake_id、review_plan_id、is_correct、review_stage_before、review_stage_after、mastery_before、mastery_after、review_time |
| 3.3 | 定义复习阶段枚举 | `ReviewStage`：阶段 0→1→2→3→4→5→6，对应间隔 0/1/2/4/7/15/30 天 |
| 3.4 | 实现状态更新规则 | 答对：阶段 +1，掌握度 +15~20；答错：阶段回退到 max(0, stage-2)，掌握度 -10~15 |
| 3.5 | 实现优先级排序策略 | 优先级 = 逾期天数 × 权重A + (100 - mastery_level) × 权重B，数值越高优先复习 |
| 3.6 | 实现每日任务生成逻辑 | 从 `next_review_time <= today` 的错题中按优先级取 top N（N = daily_limit），写入 `review_plan` |
| 3.7 | Redis 缓存当日任务 | key: `review:daily:{accountId}:{date}`，过期时间到次日凌晨 |
| 3.8 | 定时任务 | `@Scheduled(cron = "0 0 2 * * ?")` 每日凌晨 2 点为所有活跃用户预生成当日复习计划 |

- 调度算法单元测试通过 10 个以上 Case

## 六、艾宾浩斯复习调度算法核心设计

### 6.1 复习阶段与间隔

| 阶段 (stage) | 间隔 (天) | 含义 |
|:---:|:---:|------|
| 0 | 0 | 新错题，当天需复习 |
| 1 | 1 | 第 1 天后复习 |
| 2 | 2 | 第 3 天后复习 |
| 3 | 4 | 第 7 天后复习 |
| 4 | 7 | 第 14 天后复习 |
| 5 | 15 | 第 29 天后复习 |
| 6 | 30 | 第 59 天后复习（已掌握） |

### 6.2 状态更新规则

```
答对：
    stage = min(stage + 1, 6)
    mastery = min(mastery + 20, 100)
    next_review_time = now + INTERVALS[new_stage]

答错：
    stage = max(stage - 2, 0)
    mastery = max(mastery - 15, 0)
    next_review_time = now + INTERVALS[new_stage]
```

### 6.3 每日任务生成优先级公式

```
priority = overdue_days × 3.0 + (100 - mastery_level) × 0.5 + (6 - review_stage) × 2.0
```

- `overdue_days`：逾期天数（next_review_time 距今天数，≥0）
- 值越高，越优先被选入当日复习列表
- 从候选池中取 priority 最高的 N 条（N = 用户的 daily_limit）
