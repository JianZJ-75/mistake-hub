### 迭代 3：艾宾浩斯复习调度算法（第 5~6 周）

**目标**：实现核心调度算法，自动生成每日复习任务。

#### 后端

| # | 任务 | 说明 |
|---|------|------|
| 3.1 | 设计并建表 `review_plan` | id、account_id、mistake_id、planned_date、status（PENDING/COMPLETED/SKIPPED）、created_time |
| 3.2 | 设计并建表 `review_record` | id、account_id、mistake_id、review_plan_id、is_correct、review_stage_before、review_stage_after、mastery_before、mastery_after、review_time |
| 3.3 | 定义复习阶段枚举 | `ReviewStage`：阶段 0→1→2→3→4→5→6，对应间隔 0/1/2/4/7/15/30 天 |
| 3.4 | 实现状态更新规则 | 答对：阶段 +1，掌握度 +20；答错：阶段回退 max(0, stage-2)，掌握度 -15 |
| 3.5 | 实现优先级排序策略 | priority = overdue_days × 3.0 + (100 - mastery_level) × 0.5 + (6 - review_stage) × 2.0 |
| 3.6 | 实现每日任务生成逻辑 | 从 `next_review_time <= today` 的错题中按优先级取 top N（N = daily_limit） |
| 3.7 | Redis 缓存当日任务 | key: `review:daily:{accountId}:{date}`，过期时间到次日凌晨 |
| 3.8 | 定时任务 | `@Scheduled` 每日凌晨 2 点为所有活跃用户预生成当日复习计划 |

#### 前端

本迭代为纯后端算法，无前端任务。算法产出的数据将在迭代 4 的前端页面中使用。

**交付标准**：
- 调度算法单元测试通过 10 个以上 Case
- 定时任务可正确生成当日复习列表
- Redis 缓存命中时不查数据库

## 四、关键依赖路径

```
迭代1(认证+用户) ──→ 迭代2(错题) ──→ 迭代3(算法) ──→ 迭代4(复习执行)
                                                          │
                                     迭代5(统计) ←────────┘
                                          │
                                     迭代6(管理端)
                                          │
                                     迭代7(系统测试)
```

- **迭代 1→2→3→4** 是串行关键路径
- **迭代 5** 依赖迭代 4 的复习记录数据
- **迭代 6** 依赖迭代 5 的统计接口
- **迭代 7** 依赖所有前序迭代完成
- **每个迭代内**后端与前端可并行开发（后端先出 Swagger 文档，前端 Mock 并行）

## 五、核心数据库表设计预览

```sql
-- 迭代 2 新增
CREATE TABLE mistake (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY,
    account_id      BIGINT       NOT NULL COMMENT '所属用户',
    title           TEXT         NOT NULL COMMENT '题干内容',
    correct_answer  TEXT                  COMMENT '正确答案',
    error_reason    TEXT                  COMMENT '错误原因说明',
    image_url       VARCHAR(512)          COMMENT '错题图片URL',
    subject         VARCHAR(64)           COMMENT '学科',
    review_stage    INT          NOT NULL DEFAULT 0  COMMENT '复习阶段 0-6',
    mastery_level   INT          NOT NULL DEFAULT 0  COMMENT '掌握度 0-100',
    last_review_time DATETIME              COMMENT '最近复习时间',
    next_review_time DATETIME              COMMENT '下次复习时间',
    status          TINYINT      NOT NULL DEFAULT 1  COMMENT '1-有效 0-删除',
    created_time    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_time    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_account_status (account_id, status),
    INDEX idx_next_review (account_id, next_review_time)
) COMMENT '错题表';

CREATE TABLE tag (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY,
    name            VARCHAR(128) NOT NULL COMMENT '标签名称',
    type            VARCHAR(32)  NOT NULL COMMENT '类型: SUBJECT/CHAPTER/KNOWLEDGE',
    parent_id       BIGINT       DEFAULT 0 COMMENT '父标签ID，0为顶级',
    created_time    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP
) COMMENT '标签表';

CREATE TABLE mistake_tag (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY,
    mistake_id      BIGINT NOT NULL,
    tag_id          BIGINT NOT NULL,
    UNIQUE INDEX uk_mistake_tag (mistake_id, tag_id)
) COMMENT '错题-标签关联表';

-- 迭代 3 新增
CREATE TABLE review_plan (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY,
    account_id      BIGINT   NOT NULL,
    mistake_id      BIGINT   NOT NULL,
    planned_date    DATE     NOT NULL COMMENT '计划复习日期',
    status          VARCHAR(16) NOT NULL DEFAULT 'PENDING' COMMENT 'PENDING/COMPLETED/SKIPPED',
    created_time    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_account_date (account_id, planned_date)
) COMMENT '复习计划表';

CREATE TABLE review_record (
    id                   BIGINT AUTO_INCREMENT PRIMARY KEY,
    account_id           BIGINT   NOT NULL,
    mistake_id           BIGINT   NOT NULL,
    review_plan_id       BIGINT            COMMENT '关联计划ID',
    is_correct           TINYINT  NOT NULL COMMENT '1-答对 0-答错',
    review_stage_before  INT      NOT NULL COMMENT '复习前阶段',
    review_stage_after   INT      NOT NULL COMMENT '复习后阶段',
    mastery_before       INT      NOT NULL COMMENT '复习前掌握度',
    mastery_after        INT      NOT NULL COMMENT '复习后掌握度',
    review_time          DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_account_time (account_id, review_time)
) COMMENT '复习记录表';
```

## 六、艾宾浩斯复习调度算法核心设计

### 6.1 复习阶段与间隔

| 阶段 (stage) | 间隔 (天) | 含义 |
|:---:|:---:|------|
| 0 | 0 | 新错题，当天需复习 |
| 1 | 1 | 第 1 天后复习 |
| 2 | 2 | 第 3 天后复习 |
| 3 | 4 | 第 7 天后复习 |
| 4 | 7 | 第 14 天后复习 |
| 5 | 15 | 第 29 天后复习 |
| 6 | 30 | 第 59 天后复习（已掌握） |

### 6.2 状态更新规则

```
答对：
    stage = min(stage + 1, 6)
    mastery = min(mastery + 20, 100)
    next_review_time = now + INTERVALS[new_stage]

答错：
    stage = max(stage - 2, 0)
    mastery = max(mastery - 15, 0)
    next_review_time = now + INTERVALS[new_stage]
```

### 6.3 每日任务生成优先级公式

```
priority = overdue_days × 3.0 + (100 - mastery_level) × 0.5 + (6 - review_stage) × 2.0
```

- `overdue_days`：逾期天数（next_review_time 距今天数，≥0）
- 值越高，越优先被选入当日复习列表
- 从候选池中取 priority 最高的 N 条（N = 用户的 daily_limit）
