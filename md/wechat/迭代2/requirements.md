# 微信小程序迭代2实施规划 - 需求文档

基于迭代2后端已完成的错题 CRUD、标签管理和图片上传接口，实现微信小程序端的错题列表、录入、详情编辑和图片上传功能。

## Core Features

### 2.M1 错题列表页

- 替换"错题" Tab 的占位内容为实际错题列表
- 列表展示错题：题干摘要、学科标签、掌握度色标、创建时间
- 支持按学科筛选（顶部标签栏）
- 下拉刷新 + 上拉加载更多（分页）
- 右下角悬浮按钮入口跳转到录入页
- 点击列表项跳转到详情页

### 2.M2 错题录入页

- 新页面（非 Tab 页），通过 `Taro.navigateTo` 进入
- 表单字段：
  - 题干（多行文本输入）
  - 正确答案（多行文本输入）
  - 错误原因（多行文本输入）
  - 学科选择（Picker，选项来自标签树 SUBJECT 级别）
  - 标签选择（多选，来自标签树 CHAPTER/KNOWLEDGE 级别）
  - 图片上传（拍照/相册选择 → 上传 → 预览）
- 提交后返回列表页并刷新

### 2.M3 图片上传组件

- 支持拍照和从相册选择
- 选择后调用 `Taro.uploadFile` 上传到后端 `/v1/upload/image`
- 上传过程中显示 loading 状态
- 上传成功后预览图片
- 支持删除已上传的图片

### 2.M4 错题详情页

- 新页面（非 Tab 页），通过 `Taro.navigateTo` 进入
- 展示完整错题信息：题干、答案、错误原因、图片、学科、标签、复习状态
- 编辑操作：切换为编辑模式或跳转编辑页
- 删除操作：弹出确认 → 调用删除接口 → 返回列表页

## User Stories

- 作为学生，我可以在"错题"Tab 看到自己的所有错题，按时间倒序排列
- 作为学生，我可以通过顶部学科标签快速筛选某科的错题
- 作为学生，我可以下拉刷新获取最新数据，上拉加载更多历史数据
- 作为学生，我可以点击右下角按钮录入新的错题
- 作为学生，我可以拍照或从相册选图作为错题附件
- 作为学生，我可以点击某道错题查看完整详情
- 作为学生，我可以编辑或删除自己的错题

## Acceptance Criteria

- [ ] "错题" Tab 页展示实际错题列表，替换占位内容
- [ ] 列表支持下拉刷新和上拉加载更多
- [ ] 学科标签栏可筛选列表数据
- [ ] 错题录入页表单提交成功，列表页自动刷新
- [ ] 图片上传成功，预览正确显示
- [ ] 错题详情页完整展示所有字段
- [ ] 编辑保存成功，返回后列表数据更新
- [ ] 删除确认后错题消失，返回列表页

## Non-functional Requirements

- **兼容性**：支持微信基础库 3.0.0+，适配主流 iPhone / Android 机型
- **性能**：列表首页加载时间 < 2 秒；图片上传支持压缩（Taro.compressImage）
- **体验**：上传过程有 loading 反馈，网络异常有 toast 提示

## 依赖的后端接口

| 接口 | URL | 方法 | 说明 |
|------|-----|------|------|
| 错题列表 | `/v1/mistake/list` | POST | 分页查询，支持学科/标签/关键字/掌握度筛选 |
| 错题详情 | `/v1/mistake/detail` | POST | 返回完整错题信息（含标签列表） |
| 录入错题 | `/v1/mistake/add` | POST | 创建新错题 |
| 编辑错题 | `/v1/mistake/update` | POST | 更新错题信息 |
| 删除错题 | `/v1/mistake/delete` | POST | 软删除错题 |
| 上传图片 | `/v1/upload/image` | POST | multipart/form-data，返回图片 URL |
| 标签树 | `/v1/tag/tree` | POST | 返回三级标签树结构 |
