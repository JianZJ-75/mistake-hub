# 部署指南

本文档面向**零基础用户**，手把手引导你在 **Mac** 或 **Windows** 上搭建 Mistake Hub 的完整开发环境。

---

## 1. 环境概览

### 系统架构

```
┌─────────────────────┐     ┌──────────────────────────┐     ┌──────────────┐
│  浏览器 (Web 管理端)  │────→│                          │────→│   MySQL 8    │
└─────────────────────┘     │   后端 API (Spring Boot)   │     └──────────────┘
                            │                          │
┌─────────────────────┐     │   :8080                  │     ┌──────────────┐
│     微信小程序        │────→│   Swagger :18080         │────→│   Redis 7    │
└─────────────────────┘     │                          │     └──────────────┘
                            └────────────┬─────────────┘
                                         │
                                         ▼
                                 ┌──────────────┐
                                 │ 阿里云 OSS    │
                                 │  (图片存储)    │
                                 └──────────────┘
```

### 工具清单

| 工具 | 版本要求 | 用途 | 必须 |
|------|---------|------|:---:|
| Git | 任意 | 克隆代码 | 是 |
| Docker Desktop | 最新版 | 运行 MySQL 和 Redis | 是 |
| JDK | 17 | 编译运行后端 | 是 |
| Maven | 3.8+ | 后端构建 | 是 |
| Node.js | 20+ | 前端运行环境 | 是 |
| pnpm | 10+ | Web 管理端包管理 | 是 |

## 2. 安装开发工具

### 2.1 Git

**Mac**：
```bash
# Xcode Command Line Tools 自带 Git
xcode-select --install
# 或通过 Homebrew
brew install git
```

**Windows**：
1. 下载 [Git for Windows](https://git-scm.com/download/win)
2. 安装时保持默认选项，一路 Next
3. 安装完成后打开 **Git Bash** 或 **PowerShell**

**验证**：
```bash
git --version
# 输出类似：git version 2.x.x
```

### 2.2 Docker Desktop

**Mac**：
1. 下载 [Docker Desktop for Mac](https://www.docker.com/products/docker-desktop/)
   - Apple Silicon (M1/M2/M3/M4)：选择 **Apple chip** 版本
   - Intel Mac：选择 **Intel chip** 版本
2. 拖入 Applications，启动 Docker Desktop
3. 等待右上角鲸鱼图标变为稳定状态（不再动画）

**Windows**：
1. 确保已启用 **WSL2**（Windows 10 2004+ / Windows 11）：
   ```powershell
   # 以管理员身份运行 PowerShell
   wsl --install
   # 重启电脑
   ```
2. 下载 [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)
3. 安装时勾选 **Use WSL 2 instead of Hyper-V**
4. 安装完成后启动 Docker Desktop

**验证**：
```bash
docker --version
# 输出类似：Docker version 27.x.x
docker compose version
# 输出类似：Docker Compose version v2.x.x
```

### 2.3 JDK 17

**Mac**：
```bash
brew install openjdk@17
# 按提示设置 JAVA_HOME，通常需要执行：
sudo ln -sfn $(brew --prefix)/opt/openjdk@17/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk-17.jdk
```

或从 [Adoptium](https://adoptium.net/temurin/releases/?version=17) 下载 `.pkg` 安装包。

**Windows**：
1. 从 [Adoptium](https://adoptium.net/temurin/releases/?version=17) 下载 `.msi` 安装包
2. 安装时勾选 **Set JAVA_HOME variable**
3. 安装完成后**重新打开** PowerShell

**验证**：
```bash
java -version
# 输出类似：openjdk version "17.x.x"
```

### 2.4 Maven

**Mac**：
```bash
brew install maven
```

**Windows**：
1. 从 [Maven 官网](https://maven.apache.org/download.cgi) 下载 Binary zip（如 `apache-maven-3.9.x-bin.zip`）
2. 解压到 `C:\Program Files\Maven`
3. 添加环境变量：
   - `MAVEN_HOME` = `C:\Program Files\Maven\apache-maven-3.9.x`
   - 在 `Path` 中添加 `%MAVEN_HOME%\bin`
4. **重新打开** PowerShell

**验证**：
```bash
mvn -version
# 输出类似：Apache Maven 3.9.x
```

> Maven 下载依赖慢？参考 [附录 A：Maven 阿里云镜像配置](#附录-a-maven-阿里云镜像配置)。

### 2.5 Node.js

**Mac**：
```bash
brew install node@20
```

**Windows**：
1. 从 [Node.js 官网](https://nodejs.org/) 下载 LTS 版本（20.x）
2. 安装时保持默认选项

**验证**：
```bash
node --version
# 输出类似：v20.x.x
```

### 2.6 pnpm

安装 Node.js 后执行：
```bash
npm install -g pnpm@10
```

**验证**：
```bash
pnpm --version
# 输出类似：10.x.x
```

### 2.7 微信开发者工具（可选）

仅在需要调试微信小程序时安装：
1. 从 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html) 下载对应平台版本
2. 安装后使用微信扫码登录

## 4. 启动 MySQL 和 Redis

确保 Docker Desktop 已启动（状态栏图标稳定），然后在项目根目录执行：

```bash
docker compose up -d
```

首次执行会下载镜像，耐心等待。完成后验证：

```bash
# 查看容器状态（应显示 healthy）
docker ps

# 验证 MySQL
docker exec -it mysql mysql -uroot -p'mysql-1q2w3e4R' -e "SELECT 1"
# 应输出一个简单的表格，值为 1

# 验证 Redis
docker exec -it redis redis-cli -a 'redis-1q2w3e4R' ping
# 应输出：PONG
```

### 常见问题

**端口被占用**：

如果 3306 或 6379 端口已被占用，先查看占用进程：
```bash
# Mac
lsof -i :3306
# Windows PowerShell
netstat -ano | findstr :3306
```

可以停止占用进程，或修改 `docker-compose.yml` 中的端口映射（如 `"13306:3306"`），然后同步修改 `backend/src/main/resources/application.yml` 中的数据库连接地址。

**Docker Desktop 未启动**：

出现 `Cannot connect to the Docker daemon` 错误时，确保 Docker Desktop 已启动并等待初始化完成。

## 5. 初始化数据库


```powershell
Get-Content resources\db\mistake-hub.sql | docker exec -i mysql mysql -uroot -p'mysql-1q2w3e4R'
```

**Windows PowerShell**：
```powershell
Get-Content backend\src\main\resources\sql\V6__system_config_and_operation_log.sql | docker exec -i mysql mysql -uroot -p'mysql-1q2w3e4R' mistake_hub
```

## 6. 启动后端

### 6.1 配置环境变量

后端需要以下环境变量（微信和 OSS 功能可先用占位值，不影响 Web 管理端使用）：

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `WECHAT_APP_ID` | 微信小程序 AppID | `wx1234567890abcdef` |
| `WECHAT_APP_SECRET` | 微信小程序 AppSecret | `your_app_secret` |
| `OSS_ENDPOINT` | 阿里云 OSS Endpoint | `https://oss-cn-hangzhou.aliyuncs.com` |
| `OSS_ACCESS_KEY_ID` | 阿里云 OSS AccessKey ID | `your_access_key_id` |
| `OSS_ACCESS_KEY_SECRET` | 阿里云 OSS AccessKey Secret | `your_access_key_secret` |
| `OSS_BUCKET_NAME` | 阿里云 OSS Bucket 名称 | `your_bucket_name` |

**Mac / Linux**（在当前终端窗口执行）：
```bash
export WECHAT_APP_ID=placeholder
export WECHAT_APP_SECRET=placeholder
export OSS_ENDPOINT=https://oss-cn-hangzhou.aliyuncs.com
export OSS_ACCESS_KEY_ID=placeholder
export OSS_ACCESS_KEY_SECRET=placeholder
export OSS_BUCKET_NAME=placeholder
```

**Windows PowerShell**（在当前终端窗口执行）：
```powershell
$env:WECHAT_APP_ID="placeholder"
$env:WECHAT_APP_SECRET="placeholder"
$env:OSS_ENDPOINT="https://oss-cn-hangzhou.aliyuncs.com"
$env:OSS_ACCESS_KEY_ID="placeholder"
$env:OSS_ACCESS_KEY_SECRET="placeholder"
$env:OSS_BUCKET_NAME="placeholder"
```

> 这些环境变量仅在当前终端窗口生效。如需永久生效，Mac 写入 `~/.zshrc`，Windows 通过「系统属性 → 环境变量」设置。

### 6.2 构建并启动

```bash
# 先安装 BOM 模块到本地仓库
cd dependencies && mvn install -Dmaven.test.skip=true -q && cd ..

# 启动后端
cd backend && mvn spring-boot:run -Dmaven.test.skip=true
```

首次构建需要下载依赖，耐心等待。

### 6.3 验证

- 控制台输出 `Started MistakeHubBackendApplication`
- 访问 Swagger UI：http://localhost:18080/actuator/swagger-ui/index.html

### 常见问题

**环境变量未设置**：

出现 `Could not resolve placeholder 'WECHAT_APP_ID'` 错误，说明环境变量未正确设置。请确保在**同一个终端窗口**中先设置环境变量再启动后端。

**Maven 下载慢**：

参考 [附录 A：Maven 阿里云镜像配置](#附录-a-maven-阿里云镜像配置)。

**端口 8080 被占用**：

```bash
# Mac
lsof -i :8080
# Windows PowerShell
netstat -ano | findstr :8080
```

找到占用进程并停止，或修改 `backend/src/main/resources/application.yml` 中的 `server.port`。

## 8. 启动微信小程序（可选）

> 需要有效的微信小程序 AppID 和 AppSecret，并已正确配置环境变量。

**打开新的终端窗口**，执行：

```bash
cd frontend/mistake-hub-wechat
npm install
npm run dev:weapp
```

编译完成后：

1. 打开**微信开发者工具**
2. 选择「导入项目」
3. 目录选择 `frontend/mistake-hub-wechat/dist/`
4. AppID 填写你的小程序 AppID
5. 点击「导入」

### 常见问题

**dist 目录不存在**：

确保 `npm run dev:weapp` 已执行且没有报错。编译成功后 `dist/` 目录会自动生成。

**AppID 不匹配**：

微信开发者工具中的 AppID 必须与后端环境变量 `WECHAT_APP_ID` 一致。

**网络请求失败**：

在微信开发者工具中：「设置 → 项目设置 → 勾选"不校验合法域名"」。

## 9. 完整启动顺序

按以下顺序启动各服务：

| 步骤 | 服务 | 命令 | 验证 |
|:---:|------|------|------|
| 1 | MySQL + Redis | `docker compose up -d` | `docker ps` 显示 healthy |

## 10. 停止服务

```bash
# 停止后端 / 前端：在对应终端窗口按 Ctrl+C

# 停止 MySQL 和 Redis（保留数据）
docker compose down

# 停止并清除所有数据（下次需重新初始化数据库）
docker compose down -v
```

## 附录 A：Maven 阿里云镜像配置

如果 Maven 下载依赖很慢，可配置阿里云镜像加速。

编辑 Maven 配置文件：
- **Mac**：`~/.m2/settings.xml`
- **Windows**：`C:\Users\{用户名}\.m2\settings.xml`

如果文件不存在，新建并写入以下内容：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<settings>
  <mirrors>
    <mirror>
      <id>aliyun</id>
      <name>Aliyun Maven Mirror</name>
      <url>https://maven.aliyun.com/repository/public</url>
      <mirrorOf>central</mirrorOf>
    </mirror>
  </mirrors>
</settings>
```

保存后重新执行 Maven 命令即可。
